Correction lifecycle — how the 5 tables connect

1
New row inserted into semantic_field_definitions (the corrected definition).
2
Old row's superseded_by set to the new definition_id.
3
All matching rows in dataset_field_bindings have is_stale = true written in bulk.
4
A propagation_events row is created anchoring old → new definition.
5
dataset_lineage DAG is traversed (BFS from stale datasets) to find full transitive impact set.
6
One propagation_tasks row per affected dataset is bulk-inserted with computed topological_depth and blocked_by_task_ids.
7
Tasks execute in topological order. Each completion updates dataset_field_bindings (is_stale = false, new definition_id) and increments propagation_events.completed_dataset_count.
