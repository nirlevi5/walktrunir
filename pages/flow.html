Success path

  • Metadata Writer wraps all three Aurora writes in one transaction — the state is consistent before EventBridge fires.
  • DAG Traversal inserts the full task set up front so scope is visible immediately; only depth-0 tasks are enqueued first.
  • Each successful Completion Handler call promotes datasets.current_materialization_uri, increments completed_dataset_count, and releases the next topological wave to SQS — no Step Functions needed.
  • Batch path: Glue / EMR Serverless recomputes data, publishes completion via EventBridge job state change event.
  • Streaming path: consumer is paused, drained, config updated in Parameter Store, historical data is backfilled from the raw S3 archive, then live processing resumes from the drained checkpoint.
  • Rollback is a reverse DefinitionCorrected event — same pipeline, new_definition_id points back to the prior version.

Failure paths

  • Invalid request: API Gateway schema validation rejects before Lambda runs — no DB write, no EventBridge event.
  • DAG Traversal Lambda failure: EventBridge archive allows replay of DefinitionCorrected without re-running the correction itself.
  • Task retry: the failure handler increments attempt_count and enqueues a fresh start/control message below the retry threshold.
  • Task failure: after repeated failures, the task is marked failed. It is not counted as completed; its dataset keeps the previous committed materialization and direct bindings stay stale until retry or rollback succeeds.
  • Propagation failure: if any task remains failed or blocked past the retry policy, propagation_events.status = failed; the event is never shown as completed.
  • Partial rollback: Metadata Writer can scope the is_stale sweep to a single tenant_id, creating tasks only for that tenant.
